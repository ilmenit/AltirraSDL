# river_raid case study

This directory hosts the reverse-engineering project for **River Raid**
(Activision, 1982, ported to the Atari 8-bit family in 1983 by Carol
Shaw). It is structured as a worked example demonstrating how to use
AltirraBridge + Claude Code to produce a fully labelled, fully
documented, byte-perfect MADS-reassemblable disassembly of an Atari
8-bit binary, matching the depth of the
[`behind_jaggi_lines/`](../behind_jaggi_lines/) case study.

## What's in this directory

- **`PROMPT.md`** — the Claude Code prompt that drives the analysis.
  This is the only file you need to start. Open it in Claude Code,
  ensure the prerequisites listed at the top are satisfied, and tell
  the agent to begin.
- **`river_raid.xex`** — *not* shipped. River Raid is a copyrighted
  commercial game; you must provide your own legally-obtained copy
  and place it at this exact path before running the prompt. See the
  legal note below.
- **`river_raid.pyA8/`** — populated by the agent as the analysis
  progresses. Mirrors the layout of `behind_jaggi_lines/bjl.pyA8/`:
  `findings.md`, `regions.json`, `comments.json`, `xrefs.json`,
  `call_graph.json`, `notes/*.md`, `exports/*.asm`.

## How to run it

1. Place your `river_raid.xex` at `./river_raid.xex` (this directory).
2. Start the headless bridge server in another shell:
       ./AltirraBridgeServer --bridge=tcp:127.0.0.1:0
   Note the token-file path it prints to stderr.
3. Open this directory in Claude Code:
       claude code .
   The bundled `altirra-bridge` skill should auto-discover (install
   it first with `python -m altirra_bridge.install_skills` if not).
4. Tell Claude: "Read PROMPT.md and start the analysis."
5. Walk away for a few hours. Periodically check progress in
   `river_raid.pyA8/findings.md` and the `notes/` directory.

The end state is a byte-perfect MADS reassembly verified against the
SHA-256 of the original `river_raid.xex`, plus comprehensive
documentation matching the depth of `behind_jaggi_lines/`.
