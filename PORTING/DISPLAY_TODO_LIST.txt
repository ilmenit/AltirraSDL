1. CORE DISPLAY (VDVideoDisplaySDL3)
================================================================================

  [100%] VDVideoDisplaySDL3 class              — Implements IVDVideoDisplay
  [100%] PostBuffer() frame submission         — Uploads VDPixmap to SDL_Texture
  [100%] SetSource() for static content        — Handles direct pixmap display
  [100%] Present() rendering to SDL_Renderer   — Called each frame from main loop
  [100%] SDL_Texture creation and management   — Streaming texture for emulator output
  [100%] XRGB8888 pixel format                 — Primary format handled correctly

  DISPLAY.md DIVERGENCE: Doc says constructor is (SDL_Window*, SDL_Renderer*).
  Actual: constructor is (SDL_Renderer*, int w, int h). No SDL_Window param.


================================================================================
2. SCALING AND ASPECT RATIO
================================================================================

  [100%] Destination rectangle calculation     — ComputeDisplayRect() in main_sdl3.cpp.
                                                Ported from Windows ResizeDisplay() algorithm.
                                                SDL_RenderTexture uses computed dest rect.
  [100%] Aspect ratio correction              — NTSC/PAL pixel aspect ratio applied via
                                                GTIA GetPixelAspectRatio(). All 5 stretch
                                                modes fully functional.
  [100%] Integer scaling                      — Integral and IntegralPreserveAspectRatio
                                                modes snap to integer multiples with
                                                rounding-error leeway (floor(zoom * 1.0001)).
  [100%] Filter mode switching                — Point/Bilinear via UpdateScaleMode().
                                                ATUIGet/SetDisplayFilterMode() applied to
                                                SDL texture scale mode each frame.


================================================================================
3. FILTER MODES
================================================================================

  [100%] Point (nearest neighbor)             — GL_NEAREST / SDL_SCALEMODE_NEAREST
  [100%] Bilinear                             — GL_LINEAR / SDL_SCALEMODE_LINEAR
  [100%] Sharp Bilinear                       — GL backend: FEAT_SHARP shader with
                                                pixel-snap + sharpness-controlled blend.
                                                SDL_Renderer fallback: bilinear.
  [100%] Bicubic                              — GL backend: 2-pass separable cubic
                                                filter via lookup textures + FBOs.
                                                SDL_Renderer fallback: bilinear.
  [100%] Mode stored via ATUIGet/SetDisplayFilterMode()

  Note: All 5 filter modes fully implemented with the GL backend.
  Sharp bilinear and bicubic fall back to bilinear on SDL_Renderer.

5. SCREEN EFFECTS (Post-Processing)
================================================================================

  [100%] Scanline simulation                  — GL backend: GPU-rendered scanlines via
                                                scanline mask texture + FEAT_SCANLINES
                                                shader. SDL_Renderer fallback: GTIA
                                                software scanlines.
  [100%] Bloom effects                        — GL backend: Bloom V2 with 5-level FBO
                                                pyramid, downsample/upsample/compose passes.
                                                VDDComputeBloomV2Parameters() from bloom.cpp.
  [100%] Screen mask (CRT simulation)         — GL backend: aperture grille, slot mask,
                                                dot triad via VDDisplay texture generation
                                                + FEAT_DOT_MASK shader.
  [  0%] HDR output                           — SetHDREnabled() is stub; GetHDRCapability()
                                                returns NoMinidriverSupport. Deferred.
  [100%] Distortion (barrel, etc.)            — GL backend: CRT barrel distortion via
                                                FEAT_DISTORTION shader + VDDisplay-
                                                DistortionMapping.
  [  0%] Custom effect path                   — librashader runtime present but no UI.
                                                Menu item not yet added.

  Adjust Screen Effects dialog (ui_screenfx.cpp):
  [100%] Main tab - Scanline intensity, H/V distortion sliders
  [ 80%] Bloom tab - Threshold + enable. Missing: radius, direct/indirect labels
  [  0%] HDR tab - Stub only ("HDR not available" message)
  [100%] Mask tab - Screen mask type, dot pitch

  Adjust Colors dialog (ui_display.cpp — IDD_ADJUST_COLORS):
  [ 50%] Core sliders (hue/brightness/contrast/saturation/gamma) — DONE
  [ 50%] Missing: palette preview swatches, Export Palette, Palette Solver

  Note: Full shader-based screen effects require the GL backend.

================================================================================

                                                SDL_Renderer: SDL_SetRenderVSync().

                                                with error accumulator matching Windows
  [100%] NTSC / PAL / SECAM frame rates       — Exact constants from constants.h.
                                                Hardware/Broadcast/Integral frame rate
                                                modes fully supported via kFramePeriods table
  [100%] Speed modifiers                      — Speed modifier, slow motion (0.5x factor),

                                                for frame rate mode and speed. Matches
                                                Windows ATUIUpdateSpeedTiming() exactly.

                                                display refresh rate if within 2% of
                                                target. Eliminates judder on 60Hz
                                                displays with 59.92Hz NTSC content.
                                                Uses SDL_GetCurrentDisplayMode() for
                                                precise refresh rate.
