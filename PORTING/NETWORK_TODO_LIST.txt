================================================================================
3. ARCHITECTURE NOTES
================================================================================

  Uses POSIX sockets (not SDL3_net) because the socket worker requires:
  - Raw fd access for poll(), getsockname(), getpeername()
  - setsockopt for TCP_NODELAY, SO_OOBINLINE, SO_LINGER, IPV6_V6ONLY
  - shutdown() for half-close (SHUT_WR, SHUT_RD)
  - Non-blocking I/O via fcntl(O_NONBLOCK)
  - poll() for multiplexed event notification

  SDL3_net's higher-level API does not expose these socket options.
  POSIX sockets are the direct equivalent of the Winsock API used on Windows.

  Key Winsock → POSIX translations:
  - SOCKET → int, INVALID_SOCKET → -1
  - WSAEventSelect + WaitForMultipleObjects → poll() + self-pipe
  - WSAEnumNetworkEvents → poll() revents flags
  - closesocket → close, SD_SEND → SHUT_WR
  - WSAGetLastError → errno, WSAEWOULDBLOCK → EAGAIN/EWOULDBLOCK
  - GetAdaptersInfo (DNS gateway) → /proc/net/route parsing
